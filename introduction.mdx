API Wilayah Indonesia is a fully static REST API that serves complete administrative region data for Indonesia across four hierarchical levels: provinces (provinsi), regencies/cities (kabupaten/kota), districts (kecamatan), and villages (kelurahan/desa). Because the API is made up of pre-generated JSON files hosted on GitHub Pages, there is no authentication, no rate-limit complexity, and no server to maintain — just plain HTTP GET requests returning JSON.

## How it works

The API is built from CSV data files that are compiled into thousands of static JSON endpoint files using a PHP generator script. Those files are then deployed to GitHub Pages, where they are served directly as HTTP responses. There is no database, no backend server, and no runtime processing — every endpoint is a plain `.json` file.

<Steps>
  <Step title="Browse region data">
    Start by fetching the list of all provinces from `/api/provinces.json`.
  </Step>
  <Step title="Drill down the hierarchy">
    Use the province ID to fetch its regencies, then use the regency ID to fetch districts, and so on.
  </Step>
  <Step title="Fetch individual records">
    Use single-record endpoints like `/api/province/{id}.json` to retrieve details for a specific region.
  </Step>
</Steps>

## Key features

- **No authentication** — all endpoints are public and require no API keys or tokens
- **Fully static** — endpoints are pre-generated JSON files, served at CDN speed
- **Complete data** — covers all 34 provinces, hundreds of regencies, thousands of districts, and tens of thousands of villages
- **Self-hostable** — fork the repository and deploy to your own GitHub Pages for dedicated bandwidth
- **~100 million requests/month** — GitHub Pages provides 100 GB/month of bandwidth; with ~1 KB per endpoint that equals roughly 100 million requests per month
