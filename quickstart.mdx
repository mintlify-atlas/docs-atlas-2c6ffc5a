---
title: "Get started with API Wilayah Indonesia"
description: "Learn how to fetch Indonesian region data — provinces, regencies, districts, and villages — using simple GET requests in under 5 minutes."
---

This page walks you through making your first API call and traversing the full Indonesian regional hierarchy — from province down to village — in just a few steps. The API requires no setup, no account, and no API key.

<Note>
  No API key or authentication is required. All endpoints are public and respond to plain GET requests.
</Note>

<Steps>
  <Step title="Fetch the list of provinces">
    Start by requesting all 34 Indonesian provinces. Each province has an `id` and a `name`.

    <CodeGroup>

    ```bash curl
    curl https://emsifa.github.io/api-wilayah-indonesia/api/provinces.json
    ```

    ```javascript JavaScript
    const response = await fetch(
      'https://emsifa.github.io/api-wilayah-indonesia/api/provinces.json'
    );
    const provinces = await response.json();
    console.log(provinces);
    ```

    </CodeGroup>

    Example response:

    ```json
    [
      { "id": "11", "name": "ACEH" },
      { "id": "12", "name": "SUMATERA UTARA" }
    ]
    ```

    Pick a province `id` from the response — you will use it in the next step. This guide uses Aceh (`id: "11"`).
  </Step>

  <Step title="Fetch regencies for a province">
    Use the province ID to fetch all regencies (kabupaten/kota) within it. Replace `{provinceId}` with the ID from the previous step.

    <CodeGroup>

    ```bash curl
    curl https://emsifa.github.io/api-wilayah-indonesia/api/regencies/11.json
    ```

    ```javascript JavaScript
    const provinceId = '11';
    const response = await fetch(
      `https://emsifa.github.io/api-wilayah-indonesia/api/regencies/${provinceId}.json`
    );
    const regencies = await response.json();
    console.log(regencies);
    ```

    </CodeGroup>

    Example response:

    ```json
    [
      { "id": "1101", "province_id": "11", "name": "KABUPATEN SIMEULUE" },
      { "id": "1102", "province_id": "11", "name": "KABUPATEN ACEH SINGKIL" }
    ]
    ```

    Note the `id` of a regency to continue drilling down. This guide uses Aceh Selatan (`id: "1103"`).
  </Step>

  <Step title="Fetch districts for a regency">
    Use the regency ID to fetch all districts (kecamatan) within it.

    <CodeGroup>

    ```bash curl
    curl https://emsifa.github.io/api-wilayah-indonesia/api/districts/1103.json
    ```

    ```javascript JavaScript
    const regencyId = '1103';
    const response = await fetch(
      `https://emsifa.github.io/api-wilayah-indonesia/api/districts/${regencyId}.json`
    );
    const districts = await response.json();
    console.log(districts);
    ```

    </CodeGroup>

    Example response:

    ```json
    [
      { "id": "1103010", "regency_id": "1103", "name": "TRUMON" },
      { "id": "1103011", "regency_id": "1103", "name": "TRUMON TIMUR" }
    ]
    ```
  </Step>

  <Step title="Fetch villages for a district">
    Use the district ID to fetch all villages (kelurahan/desa) within it. This is the deepest level of the hierarchy.

    <CodeGroup>

    ```bash curl
    curl https://emsifa.github.io/api-wilayah-indonesia/api/villages/1103010.json
    ```

    ```javascript JavaScript
    const districtId = '1103010';
    const response = await fetch(
      `https://emsifa.github.io/api-wilayah-indonesia/api/villages/${districtId}.json`
    );
    const villages = await response.json();
    console.log(villages);
    ```

    </CodeGroup>

    Example response:

    ```json
    [
      { "id": "1103010001", "district_id": "1103010", "name": "KUTA PADANG" },
      { "id": "1103010002", "district_id": "1103010", "name": "RAKET" }
    ]
    ```

    You have now traversed the full four-level hierarchy: province → regency → district → village.
  </Step>
</Steps>

</Tip>
